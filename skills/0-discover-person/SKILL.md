# Discover Person

Captures who someone *is* — the raw identity that positioning, voice, entity maps, and content all draw from. This is **Skill 0** — it runs before everything else.

## When to Use

- Onboarding a new person/client/founder
- No existing `person_[name].md` in memory
- Existing profile feels thin or outdated

## Core Pattern

This is an **interview, not a form**. Five veins to mine — follow energy, not sequence:

1. **The path** — the turns, not the resume. How they got here.
2. **What they've built** — concrete receipts. Things shipped, created, solved.
3. **What they believe** — convictions that separate them from the default in their space.
4. **How they think** — mental models, frameworks, what they notice that others miss.
5. **What drives them** — the motivation underneath. Only capture what they share.

Synthesize into a **throughline** — the narrative thread connecting path, work, and beliefs.

## Output

Save as `person_[name].md` in memory with type `user`. Must include: throughline (narrative), credibility stack, sharp beliefs, thinking patterns, and a forward-mapping section noting implications for positioning/voice/entity map/content.

## Quality Check

Good: *"She kept getting handed the retention problem, realized the answer was always upstream in GTM, built the systems to prove it."*

Bad: *"Experienced healthcare marketing professional with expertise in retention and GTM strategy."*
